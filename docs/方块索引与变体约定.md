# 方块索引与变体约定

## 原则

| 情形 | 做法 |
| --- | --- |
| 同一玩法、仅机型 / 朝向等细差（例：高级抓取机 vs 抓取阀） | **同一 Index**，用 Data 位区分 |
| 形态或职责不同（立方体存储 vs 定向 Transfer / 抓取机） | **各自独立方块类型**（各自 Index） |
| 新立方体 / 普通方块 Index | 对齐 SCIENEW：`public static int Index = N;` + `IsIndexDynamic => false`（N 用 500+ 空号）；地形判定用 `Blocks[contents] is T` |

混用的代价：`IsXxx(value)` 分支膨胀、CSV / 掉落 / 创造栏 / 实体模板纠缠，后期拆库成本高。贴图可以共用 `Assets/Logistics.png`，不必为此合并方块类。

## 当前方块

| Index | 类 | 形态 | Face / 图集 |
| --- | --- | --- | --- |
| 551 | `AdvancedInserterBlock` | 定向 mesh；Data bit3 区分抓取机 / 抓取阀 | 模型 UV；debris 用格子 5 |
| 552 | `LogisticsStorageUnitBlock`（同 `BaseNormalBlock`：`static int` + `IsIndexDynamic => false`） | 不透明 `CubeBlock`，六面同贴图 | `GetFaceTextureSlot` → **5** |

连通 / 邻格判定（对齐宿主 `DoorBlock` / `WoodBlock`）：

```csharp
BlocksManager.Blocks[terrain.GetCellContentsFast(x, y, z)] is LogisticsStorageUnitBlock
```

`LogisticsStorageUnitBlock` **不**与未来 Transfer 方块共用 Index：存储是全向立方体容器语义，Transfer 预计是定向搬运；仅贴图格可相邻排布在同一图集。
